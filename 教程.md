## 一、Makefile 编译指令

### 1.1 查看所有可用板名

```bash
make help
```

输出 `supported boards are: ...` 后面是所有可编译的 board 名称。

### 1.2 编译单个板固件

```bash
make <board_name>
```

**示例**：

```bash
make 60              # 编译 VESC 6 板固件
make 60_mk5          # 编译 VESC 6 MK5
make 60_mk6          # 编译 VESC 6 MK6
make stormcore_60d   # 编译 Stormcore 60D
```

**规则**：`make <name>` → 搜索 `hw_<name>.h` → 找到文件所在的目录 → 用该目录下的 `hw_xxx_core.h`/`hw_xxx_core.c` 编译。

### 1.3 清理编译产物

```bash
make fw_<board>_clean
# 例如: make fw_60_clean
```

清理后 `build/<board>/` 目录会被删除，下次编译会从头构建。

### 1.4 编译无电流限制版本

在板名后加 `_no_limits`：

```bash
make 60_no_limits
```

### 1.5 编译所有固件

```bash
make all_fw          # 编译所有板
make all_fw_clean    # 清理所有板
```

### 1.6 打包固件（打包到 package 目录）

```bash
make all_fw_package
```

调用 `package_firmware.py` 把 `build/` 下的 `.bin` 文件整理到 `package/` 目录，生成 `res_fw.qrc` 供 VESC Tool 使用。

### 1.7 查看编译详情

加 `V=1` 显示完整编译命令：

```bash
make 60 V=1
```

### 1.8 常用板名速查

| Make 命令 | 对应硬件 | hwconf 目录 |
|-----------|----------|-------------|
| `make 60` | VESC 6 | `hwconf/trampa/vesc6/` |
| `make 60_mk3` | VESC 6 MK3 | `hwconf/trampa/vesc6/mk3/` |
| `make 60_mk5` | VESC 6 MK5 | `hwconf/trampa/vesc6/mk5/` |
| `make 60_mk6` | VESC 6 MK6 | `hwconf/trampa/vesc6/mk6/` |
| `make 75_300` | VESC 75/300 | `hwconf/trampa/75_300/` |
| `make stormcore_60d` | StormCore 60D | `hwconf/stormcore/100d/` |
| `make a50s_v22_6s` | A50S V2.2 6S | `hwconf/teamtriforceuk/a50s_v22/` |

## 二、Python 打包脚本 (`package_firmware.py`)

### 2.1 脚本作用

把 `build/<board>/<board>.bin` 文件按分类复制到 `package/` 目录，生成 QRC 资源文件。

### 2.2 脚本语法

```python
package_dict["VESC Tool 中显示的固件组名"] = [
    ['make时的board名', '输出bin文件名'],
    ['board名_no_limits', 'VESC_default_no_hw_limits.bin'],
]
```

**结构说明**：
- `package_dict` 是一个 Python 字典（dict）
- **Key**：字符串，VESC Tool 固件标签页中显示的分组名称
- **Value**：列表的列表，每个子列表包含 2 个元素：
  - 第 1 个元素：`make` 时使用的 board 名（必须与 `make 60` 中的 `60` 一致）
  - 第 2 个元素：输出到 `package/` 的 `.bin` 文件名

### 2.3 使用默认文件名常量

```python
default_name = "VESC_default.bin"               # 默认版本
no_limits_name = "VESC_default_no_hw_limits.bin" # 无限制版本
```

### 2.4 示例

```python
# VESC 6 默认版本和无限制版本
package_dict["60_MK6"] = [
    ['60_mk6', default_name],
    ['60_mk6_no_limits', no_limits_name],
]

# 自定义输出文件名
package_dict["410_o_411_o_412"] = [
    ['410', default_name],
    ['410_0005ohm', 'VESC_0005ohm.bin'],
    ['410_005ohm', 'VESC_005ohm.bin'],
]
```

### 2.5 添加新板的步骤

1. 在 `package_firmware.py` 的 `package_dict` 中添加条目：
   ```python
   package_dict["你的板名"] = [['make使用的board名', default_name]]
   ```

2. 先编译生成 bin：
   ```bash
   make <board名>
   ```

3. 打包：
   ```bash
   make all_fw_package
   ```

### 2.6 运行方式

```bash
# 自动打包（从 build/ 复制到 package/）
python package_firmware.py

# 打印所有需要 make 的 board 名（用于批量编译）
python package_firmware.py make_targets
# 输出类似: 60 60_mk3 60_mk5 stormcore_60d ...
```

## 三、电流采样极性修改

### 3.1 修改方法

在对应 board 的硬件配置头文件中添加或取消注释：

```c
#define INVERTED_SHUNT_POLARITY
```

**文件位置示例**：
- VESC 6: `hwconf/trampa/vesc6/hw_60_core.h`
- StormCore: `hwconf/stormcore/100d/hw_stormcore_100d_core.h`

### 3.2 原理

- `hwconf/hw.h:337-406` 定义了 `GET_CURRENT1/2/3` 宏
- **未定义** `INVERTED_SHUNT_POLARITY` 时：`GET_CURRENT1() = (float)ADC_Value[ADC_IND_CURR1]`
- **定义** `INVERTED_SHUNT_POLARITY` 后：`GET_CURRENT1() = 4095.0 - (float)ADC_Value[ADC_IND_CURR1]`

### 3.3 其他使用此宏的板

```
hwconf/vesc/str365/hw_str365_core.h        — 已启用
hwconf/vesc/minim/hw_minim_core.h           — 已启用
hwconf/vesc/maxim/hw_maxim_core.h           — 已启用
hwconf/makerx/hw_go_foc_m100.h              — 已启用
hwconf/teamtriforceuk/a200s_v4/v41/hw_a200s_v41_core.h — 已启用
```

## 四、方向反转

### 4.1 修改方法

通过 VESC Tool 设置 `m_invert_direction` 为 `true`。

```c
// motor/mc_interface.c:52
#define DIR_MULT  (motor_now()->m_conf.m_invert_direction ? -1.0 : 1.0)
```

### 4.2 影响范围

设为 `true` 后，以下全部取反：
- RPM（转速）
- 电流（directional current）
- 位置（position）
- PWM duty cycle
- 编码器方向

## 五、电阻 (Resistance) 测量

电阻本身无极性概念，通过 FOC 自动测量获得：

- `measure_res [current]` — 锁电机测量电阻
- `measure_res_ind` — 同时测量电阻和电感
- 测量结果存入 `mc_configuration.foc_motor_r`
