## 一、Makefile 编译指令

### 1.1 查看所有可用板名

```bash
make help
```

输出 `supported boards are: ...` 后面是所有可编译的 board 名称。

### 1.2 编译单个板固件

```bash
make <board_name>
```

**示例**：

```bash
make 60              # 编译 VESC 6 板固件
make 60_mk5          # 编译 VESC 6 MK5
make 60_mk6          # 编译 VESC 6 MK6
make stormcore_60d   # 编译 Stormcore 60D
```

**规则**：`make <name>` → 搜索 `hw_<name>.h` → 找到文件所在的目录 → 用该目录下的 `hw_xxx_core.h`/`hw_xxx_core.c` 编译。

### 1.3 清理编译产物

```bash
make fw_<board>_clean
# 例如: make fw_60_clean
```

清理后 `build/<board>/` 目录会被删除，下次编译会从头构建。

### 1.4 编译无电流限制版本

在板名后加 `_no_limits`：

```bash
make 60_no_limits
```

### 1.5 编译所有固件

```bash
make all_fw          # 编译所有板
make all_fw_clean    # 清理所有板
```

### 1.6 打包固件（打包到 package 目录）

```bash
make all_fw_package
```

调用 `package_firmware.py` 把 `build/` 下的 `.bin` 文件整理到 `package/` 目录，生成 `res_fw.qrc` 供 VESC Tool 使用。

### 1.7 查看编译详情

加 `V=1` 显示完整编译命令：

```bash
make 60 V=1
```

### 1.8 常用板名速查

| Make 命令 | 对应硬件 | hwconf 目录 |
|-----------|----------|-------------|
| `make 60` | VESC 6 | `hwconf/trampa/vesc6/` |
| `make 60_mk3` | VESC 6 MK3 | `hwconf/trampa/vesc6/mk3/` |
| `make 60_mk5` | VESC 6 MK5 | `hwconf/trampa/vesc6/mk5/` |
| `make 60_mk6` | VESC 6 MK6 | `hwconf/trampa/vesc6/mk6/` |
| `make 75_300` | VESC 75/300 | `hwconf/trampa/75_300/` |
| `make stormcore_60d` | StormCore 60D | `hwconf/stormcore/100d/` |
| `make a50s_v22_6s` | A50S V2.2 6S | `hwconf/teamtriforceuk/a50s_v22/` |

## 二、Python 打包脚本 (`package_firmware.py`)

### 2.1 脚本作用

把 `build/<board>/<board>.bin` 文件按分类复制到 `package/` 目录，生成 QRC 资源文件。

### 2.2 脚本语法

```python
package_dict["VESC Tool 中显示的固件组名"] = [
    ['make时的board名', '输出bin文件名'],
    ['board名_no_limits', 'VESC_default_no_hw_limits.bin'],
]
```

**结构说明**：
- `package_dict` 是一个 Python 字典（dict）
- **Key**：字符串，VESC Tool 固件标签页中显示的分组名称
- **Value**：列表的列表，每个子列表包含 2 个元素：
  - 第 1 个元素：`make` 时使用的 board 名（必须与 `make 60` 中的 `60` 一致）
  - 第 2 个元素：输出到 `package/` 的 `.bin` 文件名

### 2.3 使用默认文件名常量

```python
default_name = "VESC_default.bin"               # 默认版本
no_limits_name = "VESC_default_no_hw_limits.bin" # 无限制版本
```

### 2.4 示例

```python
# VESC 6 默认版本和无限制版本
package_dict["60_MK6"] = [
    ['60_mk6', default_name],
    ['60_mk6_no_limits', no_limits_name],
]

# 自定义输出文件名
package_dict["410_o_411_o_412"] = [
    ['410', default_name],
    ['410_0005ohm', 'VESC_0005ohm.bin'],
    ['410_005ohm', 'VESC_005ohm.bin'],
]
```

### 2.5 添加新板的步骤

1. 在 `package_firmware.py` 的 `package_dict` 中添加条目：
   ```python
   package_dict["你的板名"] = [['make使用的board名', default_name]]
   ```

2. 先编译生成 bin：
   ```bash
   make <board名>
   ```

3. 打包：
   ```bash
   make all_fw_package
   ```

### 2.6 运行方式

```bash
# 自动打包（从 build/ 复制到 package/）
python package_firmware.py

# 打印所有需要 make 的 board 名（用于批量编译）
python package_firmware.py make_targets
# 输出类似: 60 60_mk3 60_mk5 stormcore_60d ...
```

## 三、电流采样极性修改

### 3.1 修改方法

在对应 board 的硬件配置头文件中添加或取消注释：

```c
#define INVERTED_SHUNT_POLARITY
```

**文件位置示例**：
- VESC 6: `hwconf/trampa/vesc6/hw_60_core.h`
- StormCore: `hwconf/stormcore/100d/hw_stormcore_100d_core.h`

### 3.2 原理

- `hwconf/hw.h:337-406` 定义了 `GET_CURRENT1/2/3` 宏
- **未定义** `INVERTED_SHUNT_POLARITY` 时：`GET_CURRENT1() = (float)ADC_Value[ADC_IND_CURR1]`
- **定义** `INVERTED_SHUNT_POLARITY` 后：`GET_CURRENT1() = 4095.0 - (float)ADC_Value[ADC_IND_CURR1]`

### 3.3 其他使用此宏的板

```
hwconf/vesc/str365/hw_str365_core.h        — 已启用
hwconf/vesc/minim/hw_minim_core.h           — 已启用
hwconf/vesc/maxim/hw_maxim_core.h           — 已启用
hwconf/makerx/hw_go_foc_m100.h              — 已启用
hwconf/teamtriforceuk/a200s_v4/v41/hw_a200s_v41_core.h — 已启用
```

## 四、方向反转

### 4.1 修改方法

通过 VESC Tool 设置 `m_invert_direction` 为 `true`。

```c
// motor/mc_interface.c:52
#define DIR_MULT  (motor_now()->m_conf.m_invert_direction ? -1.0 : 1.0)
```

### 4.2 影响范围

设为 `true` 后，以下全部取反：
- RPM（转速）
- 电流（directional current）
- 位置（position）
- PWM duty cycle
- 编码器方向

## 七、死区时间 (Dead Time)

### 7.1 什么是死区时间

死区时间是指在上桥臂和下桥臂 MOSFET 切换导通时，**两个管子都保持关断的一小段时间**。

以半桥电路为例：上管和下管不能同时导通，否则电源直接对地短路（直通/shoot-through），会瞬间烧毁 MOSFET。由于 MOSFET 存在开关延迟（turn-on/turn-off time），即使软件同时关断一个、开启另一个，也会出现短暂的重叠导通。死区时间就是在控制信号中插入一段"安全间隔"，确保先关断的管子完全关断后，再开启另一个管子。

```
上管 PWM:  ──‾‾‾‾──────‾‾‾‾──────
            ───▒▒▒▒▒▒▒▒──────▒▒▒▒▒▒▒▒─  ← 死区 (两管都关断)
下管 PWM:      ──‾‾‾‾──────‾‾‾‾──
```

### 7.2 死区时间的影响

| 死区过大 | 死区过小 |
|----------|----------|
| 有效导通时间减少，扭矩下降 | 直通风险增加，可能烧毁硬件 |
| 低转速时电流波形畸变明显 | 若开关速度足够快，可以安全使用小死区 |
| 效率降低、发热增加 | |

**经验值**：多数 BLDC 电调的死区时间在 **200ns ~ 1000ns** 之间，需根据具体 MOSFET 的开关特性和驱动电路延迟来调整。

### 7.3 固件中的死区配置

**默认定义位置**：`motor/mc_interface.h:148-149`

```c
#ifndef HW_DEAD_TIME_NSEC
#define HW_DEAD_TIME_NSEC     850.0    // Dead time (单位: ns)
#endif
```

各硬件配置板可以通过定义 `HW_DEAD_TIME_NSEC` 来覆盖默认值，例如在 `hw_xxx_core.h` 中添加：

```c
#define HW_DEAD_TIME_NSEC     500.0    // 自定义死区时间 (ns)
```

### 7.4 死区时间如何写入寄存器

STM32 的定时器通过 `BDTR` 寄存器的 `DTG[7:0]` 位配置死区时间。代码在 `conf_general.c:903` 中的 `conf_general_calculate_deadtime()` 函数里，将纳秒值转换为寄存器值：

```c
uint8_t conf_general_calculate_deadtime(float deadtime_ns, float core_clock_freq) {
    float timebase = 1.0 / (core_clock_freq / 1000000.0) * 1000.0;
    // 根据目标死区时间自动选择 4 种编码模式之一
    // 模式1: DTG × timebase              (0 ~ 127 × t)
    // 模式2: (64 + DTG[5:0]) × 2t       (64 × t ~ 254 × t)
    // 模式3: (32 + DTG[5:0]) × 8t       (256 × t ~ 768 × t)
    // 模式4: (32 + DTG[5:0]) × 16t      (512 × t ~ 1024 × t)
}
```

写入位置：`motor/mcpwm_foc.c:239`（FOC 模式）和 `motor/mcpwm.c:278`（方波模式）

```c
TIM_BDTRInitStructure.TIM_DeadTime = conf_general_calculate_deadtime(HW_DEAD_TIME_NSEC, SYSTEM_CORE_CLOCK);
```

### 7.5 死区补偿 (Deadtime Compensation)

除了插入硬件死区，固件还在 FOC 控制循环中进行**死区补偿**（`mcpwm_foc.c:5081`）。

死区会导致实际输出电压与期望电压产生偏差（尤其在低转速、大电流时）。补偿方法是根据相电流方向对 PWM 占空比做微调，抵消死区造成的电压损失。具体在代码中体现为对 sin/cos 相的偏移修正：

```c
// Deadtime compensation
float s = state_m->phase_sin;
float c = state_m->phase_cos;
const float i_alpha_filter = c * state_m->id_filter - s * state_m->iq_filter;
const float i_beta_filter = c * state_m->iq_filter + s * state_m->id_filter;
```

### 7.6 修改死区时间的步骤

1. 在对应硬件配置文件中修改或定义 `HW_DEAD_TIME_NSEC`（或在 `mc_interface.h` 修改默认值）
2. 重新编译固件：`make <board_name>`
3. 通过 VESC Tool 刷写新固件
4. 观察电机运行状态，必要时微调
